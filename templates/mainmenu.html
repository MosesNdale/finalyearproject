This is the main menu of the personalised transport planner

Click any buttons on the navigation bar at the top of this website to search or click here to start.

This project is about combating climate change and congestion in public transports, the statement of this problem is the use of the REST API from TFL to be leveraged for providing more specific and personalised information for people who live in London, which is the capital city of England or within the United Kingdom. The source of the information from TFL is Powered by TfL Open Data.
To read more about TFL click here.

Note: This personalised transport planner website may require internet to use when browsing on your device!
About us: This website project about the personalised transport planner was made using python, jupyter notebook, Flask and HTML as web pages within jupyter notebook. You can click on the drop-down navigation buttons to start browsing as this website application runs on the local host.
Caution: some website pages may contain repeated results after pressing the button to submit your response on the HTML form. Please enter the values in the field of the forms to be completed in the english langauage. If the result returns empty or contains null values from a HTML form, this means that the data contains errors or it is unavailable to be provided. The results from the HTML form is automatically in light mode after entering the valid values on the fields and pressing the submit button, change to dark mode by clicking the button on the top of the website beneath the navigation bar if you prefer to read the information there.
The Rest API from TFL was mostly used to build this website, to read more about this information click here and some parts of the API from London air was used to be included on this website, to read more about this information click here. This website was written in human-readable format as plain language for the general public as a stakeholder to understand the text and machine-readable format as the JSON containing the parameters as keys and values from TFL and London Air for the developers or people who would want to understand the code that is written as the API.
To read about the frequently asked questions from TFL about the rest API on its website, click here.
